# tagma-sdk Test Suite

## Directory Structure

```text
tests/
  cases/        # YAML sample definitions
  helpers/      # Small scripts for command / hook / output_check
  workspaces/   # Per-sample sandbox working directories
  unit/         # Unit tests (bun test)
  run-sample.ts # Single entry point (supports auto-approve)
```

## Usage

```bash
# List all samples
bun tests/run-sample.ts --list

# Run one or more samples
bun tests/run-sample.ts 01-command-smoke 02-manual-ignore

# Local-only samples (no external AI CLI required)
bun tests/run-sample.ts --local

# AI-driven samples (requires claude-code / codex CLI login)
bun tests/run-sample.ts --ai

# All samples
bun tests/run-sample.ts --all

# PipelineRunner + onEvent + abort tests
bun tests/run-sample.ts --extras

# Unit tests
bun test
```

> `manual trigger` is auto-approved/rejected by `tests/run-sample.ts` by default, so it won't block.

## Workspace Output Locations

After running a sample, common artifacts are found at:

- `.tagma/logs/` — SDK runtime logs
- `.tagma-tests/generated/` — auxiliary files generated by command tasks
- `.tagma-tests/output/` — task `output:` written output
- `.tagma-tests/hook-events.log` — hook event log

## Recommended Order

```bash
# 1. Run unit tests first
bun test

# 2. Local fast samples
bun tests/run-sample.ts --local

# 3. PipelineRunner extras
bun tests/run-sample.ts --extras

# 4. AI-driven samples (requires CLI login)
bun tests/run-sample.ts --ai
```
